## Key Decisions / Conclusions
- Decision: Use Electron + React + TypeScript for the MVP. — Reason: It provides a true desktop shell with native folder access and still allows a fast modern UI implementation.
- Decision: Treat Spotify as a metadata/library source, not a playback source. — Reason: The app needed local-file deck playback, and Spotify policy/API constraints make metadata import the safer MVP path.
- Decision: Compute BPM, key, Camelot key, waveform, and energy locally. — Reason: The current product direction should not assume Spotify will provide all DJ analysis fields reliably.
- Decision: Merge Spotify tracks with local files by ISRC first and normalized title/artist second. — Reason: This gives a practical unified library without requiring a full external database.
- Decision: Build AI features as transition scoring, smart crates, and set-arc guidance. — Reason: These features make the MVP feel more like an “assistant DJ” without needing an external AI service.
- Decision: Prefer desktop-first layout and control density over a simplified consumer music UI. — Reason: The user explicitly asked for Serato / Pioneer-style DJ software rather than a casual browser music app.

## Implementation Details
- App stack: Electron 38, React 19, Vite 7, TypeScript 5, `music-metadata` for local tag parsing.
- Dev command: `npm run dev` starts Vite and then Electron once the renderer is available on `http://127.0.0.1:5173`.
- Build command: `npm run build` runs `tsc --noEmit && vite build`.
- Spotify config:
  - Redirect URI: `http://127.0.0.1:47832/spotify/callback`
  - Auth flow: Authorization Code with PKCE
  - Imported scopes: `user-library-read`, `playlist-read-private`, `playlist-read-collaborative`, `user-read-private`
- Desktop IPC surface from preload:
  - `selectMusicFolder`
  - `scanMusicFolder`
  - `readAudioFile`
  - `openExternal`
  - `spotifyLogin`
  - `spotifySync`
  - `spotifyLogout`
  - `getSpotifyRedirectUri`
- Local scan behavior:
  - Supported audio extensions: `.mp3`, `.m4a`, `.aac`, `.wav`, `.aiff`, `.aif`, `.flac`, `.ogg`, `.opus`
  - Hidden files/folders are skipped during recursive scan.
  - Metadata parse falls back to filename-derived titles when tags are missing.
- Analysis behavior:
  - Audio is mixed to mono and downsampled to `11025 Hz`.
  - BPM estimation uses an onset envelope and lag scoring window.
  - Key estimation uses chroma accumulation plus major/minor key profiles and Camelot conversion.
  - Energy is derived from RMS amplitude and normalized to a 0-100 scale.
  - Waveforms are stored as 180 peak buckets for UI rendering.
- Renderer behavior:
  - Local files are cached as `ArrayBuffer` instances and converted to blob URLs for playback.
  - Two decks maintain play state, current time, volume, playback rate, cue points, and sync state.
  - The analysis queue automatically processes playable local tracks that still need BPM/key/energy/waveform data.
  - The transition coach scores candidates by BPM gap, harmonic compatibility, energy movement, and tag overlap.
  - Smart crates include Warm Up, Peak Pressure, Harmonic Ladder, and Reset Lane.
- Verification performed:
  - `npm install`
  - `npm run build`
